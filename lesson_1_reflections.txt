Ter acesso a um histórico de alterações facilitaria muito a vida de um programador, tornando seu serviço muito mais eficiente, economizando tempo e achando facilmente erros de programação que dificilmente seriam vistos.

What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?

O programador tem um controle sobre as versões anteriores e poder comparar se algo estava funcionando antes ou não. Assim o mesmo tem a possibilidade de resgatar as versões anteriores.
O contra é que pode acontecer do programador se esquecer de salvar, por algum motivo de força maior, perder tudo que já fez.

Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

O Github foi desenvolvido para programadores, então é extremamente necessário poder salvar vários arquivos em um commit, pois as vezes é usado mais de um arquivo durante a criação de um projeto então é útil que o Git trate tais arquivos em conjunto.

How can you use the commands git log and git diff to view the history of files?

O comando git log é responsável por você ver todos os commits feitos,
O comando git diff para ver as diferenças entre os dois commits feitos.
